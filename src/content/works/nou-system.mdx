## Overview

能楽部の運営負担を解消するために作った、練習シフト自動生成 Web アプリ。能楽部ではシテ・ワキ・地謡・楽器など複数パートがあり、毎週2回、メール出席確認 45 分 + シフト作成 2 時間を手作業で繰り返していた。

**導入後:**
- メール出席確認 → Web 上での出席登録
- シフト作成 → 出席とパート登録から自動生成
- 管理者は **レビューと軽微な修正だけ** で完結

## Numbers

| 指標 | 数値 |

## My role

要件定義 → 設計 → 実装 → 運用改善まで、学生チームで完結させたうちのコア開発を担当。特に:
- 制約(パート定員・優先度・重複禁止)を満たすスケジューリングロジック
- 本番環境への Kubernetes デプロイ(namespace/ingress/secret/postgres/nginx/backup 分離)
- テスト駆動開発での品質担保(大手製造業のプロエンジニアからコードレビューを受けた経験を反映)

## Architecture

- Backend: Python / FastAPI / PostgreSQL
- Frontend: React / TypeScript
- Infra: Kubernetes + Nginx + 自動バックアップ
- Dev/Prod を `.env` と `docker-compose.prod.yml` で分離

## What this proved

学生だけで **要件定義から本番運用まで** 一気通貫できることを示した社会実装。学内ラボプロジェクトのハッカソン優勝やベンチャーコンテスト受賞につながった根っこの実績。
