## Overview

自宅の物理ランプを「LLM から直接操作できる」状態にするために、ハードウェア・ファームウェア・MCP サーバー・公開基盤まで全部自作した。end-to-end で:

```
claude.ai  ─(MCP / SSE)→  Cloudflare Tunnel
                            ↓
                         MCP Server (FastMCP, Docker)
                            ↓
                         ESP32 (HTTP)
                            ↓
                         🔦 Lamp
```

## Firmware (C++ / ESP32)

- Arduino framework on PlatformIO(`esp32dev`, `huge_app.csv`)
- **外部HTTPサーバーライブラリを使わず、素の `WiFiServer` / `WiFiClient` で HTTP/1.1 レスポンダを自作**
- BLE 広告 + WiFi 並列
- 静的 IP 設定、自動再接続

## MCP Server (Python / FastMCP)

```python
# stdio モード(Claude Code ローカル)
python server.py

# HTTP/SSE モード(claude.ai リモート)
BT_LIGHT_HOST=192.168.0.43 MCP_API_KEY=secret python server.py --sse
```

- FastMCP で ESP32 HTTP API をラップ
- OAuth トークンセッション管理
- Docker Compose でコンテナ常駐
- ログは `logs/mcp.log` に保存

## Public exposure

Cloudflare Tunnel(`cloudflared`)で自宅 NAT を越えて `*.trycloudflare.com` に公開。claude.ai からも(認証付きで)ランプを操作できる。

## Why this matters

2026 年の LLM エコシステムで MCP が標準化されつつあるが、**物理デバイスに MCP を挿せてる作品は相当少ない**。ハードの境界 → ネットワーク境界 → LLM 境界の3つをまたいで動くフルスタック IoT × AI の実演。
